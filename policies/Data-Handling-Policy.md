# Data Handling Policy (AIC)

## General Principle
AIC commits to transparency in data handling.  
Only necessary data is collected and used for community operations.

## Data Classification
1. **Public Data**  
   - Code, technical documentation, commit history.  
   - Basic logs for transparency (e.g., commit counts, contributor activity).  
   - Public metrics (e.g., issues, pull requests).

2. **Confidential Data**  
   - Internal governance discussions not yet public.  
   - Detailed financial records on OCE (beyond published reports).  
   - Logs or metrics related to system security.

## Access Rights
- Founder has full access.  
- Contributors access only public data.  
- Confidential data is shared only with explicit Founder approval.

## Commitments
- No personal data stored beyond what is strictly necessary.  
- No data used for commercial purposes.  
- All public data remains auditable once disclosed.
