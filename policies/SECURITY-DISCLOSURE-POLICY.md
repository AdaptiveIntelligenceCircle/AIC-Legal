# Security Disclosure Policy (Responsible Disclosure)

**Version 1.0** — April 2026  
**Adaptive Intelligence Circle (AIC) & Human Meaning Network (HMN)**

## 1. Purpose
We take security seriously. This policy defines how security vulnerabilities should be reported and handled responsibly, balancing transparency with user safety.

## 2. Scope
This policy applies to all repositories under the Adaptive Intelligence Circle organization, especially core modules (IBCS, Adaptive OS, SSI, Recovery, etc.).

## 3. Reporting a Vulnerability

**Preferred method**: Email to security@adaptiveintelligencecircle.org (or the Founder's public email if not set up yet).

**Please include**:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (if available)
- Your name/handle (optional)

We encourage researchers to follow responsible disclosure: do not publicly disclose the vulnerability until we have had reasonable time to investigate and patch.

## 4. Response Timeline
- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 7 days
- **Patch release target**: Within 30–90 days depending on severity
- Critical vulnerabilities (RCE, kernel-level, affecting many users) will be prioritized.

## 5. Safe Harbor
Anyone who follows this policy will not face legal action from AIC for their security research, provided they:
- Do not exploit the vulnerability beyond necessary testing
- Do not access or modify data of other users
- Do not publicly disclose before coordinated disclosure

## 6. Transparency
Once a vulnerability is fixed, we will publish a security advisory (unless doing so would cause significant harm).

---

**Commitment**: Security is part of our ethical-from-kernel principle.
